## List

The `vc list` command, which can be shortened to `vc ls`, is used to provide a list of your deployments along with information about them.

### Basic Usage

<Snippet dark text="vc list" />
<Caption>Using the <InlineCode>vc list</InlineCode> command to retrieve information about multiple deployments.</Caption>

### Extended Usage

<Snippet dark text="vc list [project-name]" />
<Caption>Using the <InlineCode>vc list</InlineCode> command to retrieve information about deployments for a specific project.</Caption>

<Snippet dark text="vc list [project-name] [--meta foo=bar]" />
<Caption>Using the <InlineCode>vc list</InlineCode> command to retrieve information about deployments filtered by metadata.</Caption>

### Global Options

The following [global options](#options) can be passed when using the `vc list` command:

- `--debug`
- `--global-config`
- `--help`
- `--local-config`
- `--scope`
- `--token`

For more information on global options and their usage, refer to the [options section](#options).

### Unique Options

These are options that only apply to the `vc list` command, therefore, more information is provided.

#### Listen

The `--meta` option, shorthand `-m`, can be used to filter results based on project metadata.
